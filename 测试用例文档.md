# CI.AI 端到端测试用例文档

> **项目**：CI.AI — AI 视频全流程自动化创作平台  
> **API 前缀**：`/api/v1`  
> **认证方式**：JWT Bearer Token（`Authorization: Bearer <token>`）  
> **测试账号**：`admin / admin123`

---

## 一、用户认证

### TC-1.1 正常登录

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-1.1 |
| **用例名称** | 正常登录 |
| **前置条件** | 用户 `admin` 已注册 |
| **请求** | `POST /api/v1/auth/login` |
| **请求体** | `{ "username": "admin", "password": "admin123" }` |
| **预期响应** | `code=0`，`data.access_token` 非空，`data.user.username == "admin"` |
| **验证点** | 1. 返回 `access_token` 非空字符串<br>2. `token_type == "bearer"`<br>3. `user.role` 为 `"user"` 或 `"admin"`<br>4. `user.credits >= 0` |
| **后续操作** | 保存 `access_token`，后续所有请求携带 `Authorization: Bearer <token>` |

## 二、角色管理

### TC-2.1 创建角色

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-2.1 |
| **用例名称** | 正常创建角色 |
| **前置条件** | 已登录 |
| **请求** | `POST /api/v1/characters`（multipart/form-data） |
| **请求参数** | `name="测试角色"`, `description="一个赛博朋克风格的少女"`, `visual_prompt="cyberpunk girl, blue hair, neon eyes, futuristic outfit"` |
| **预期响应** | `code=0`，返回 `CharacterOut` 对象 |
| **验证点** | 1. `data.id` 为正整数<br>2. `data.name == "测试角色"`<br>3. `data.visual_prompt` 非空<br>4. `data.views` 为空数组 |
| **后续操作** | 保存 `character_id` |

### TC-2.2 生成角色视图

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-2.2 |
| **用例名称** | AI 生成角色视图 |
| **前置条件** | TC-2.1 通过，已获取 `character_id` |
| **请求** | `POST /api/v1/characters/{character_id}/generate-views` |
| **请求体** | `{ "count": 3, "view_types": ["front", "side", "expression"] }` |
| **预期响应** | `code=0`，`message` 包含成功提示 |
| **验证点** | 1. 接口返回成功<br>2. 轮询 `GET /api/v1/characters/{character_id}`，直到 `data.views.length >= 3`<br>3. 每个 view 的 `image_url` 非空且可访问<br>4. `view_type` 分别为 `front`、`side`、`expression` |
| **超时** | 最长等待 120 秒 |

### TC-2.3 新增角色视图（手动上传）

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-2.3 |
| **用例名称** | 手动新增角色视图 |
| **前置条件** | TC-2.1 通过 |
| **请求** | `POST /api/v1/characters/{character_id}/generate-views` |
| **请求体** | `{ "count": 1, "view_types": ["action"] }` |
| **预期响应** | `code=0` |
| **验证点** | 1. 接口返回成功<br>2. 再次获取角色详情，`views` 中新增了 `view_type="action"` 的视图<br>3. 总视图数 = 之前数量 + 1 |

### TC-2.4 删除角色视图

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-2.4 |
| **用例名称** | 删除角色视图 |
| **前置条件** | TC-2.2 或 TC-2.3 通过，角色至少有 1 个视图 |
| **请求** | `DELETE /api/v1/characters/{character_id}/views/{view_id}` |
| **预期响应** | `code=0`，`message` 包含删除成功提示 |
| **验证点** | 1. 接口返回成功<br>2. 再次获取角色详情，`views` 中不包含已删除的 `view_id`<br>3. 总视图数 = 之前数量 - 1 |

## 五、项目全流程

### TC-5.1 创建项目

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.1 |
| **用例名称** | 正常创建项目 |
| **前置条件** | 已登录，TC-4.1 风格已创建（获得 `style_id`） |
| **请求** | `POST /api/v1/projects` |
| **请求体** | `{ "name": "赛博少女的一天", "description": "赛博朋克世界中一个蓝发少女的日常生活", "style_id": <style_id>, "shots_per_image": 2 }` |
| **预期响应** | `code=0`，返回 `ProjectOut` 对象 |
| **验证点** | 1. `data.id` 为正整数<br>2. `data.name == "赛博少女的一天"`<br>3. `data.status == "draft"`<br>4. `data.style.id == <style_id>`<br>5. `data.shots_per_image == 2` |
| **后续操作** | 保存 `project_id` |

### TC-5.2 AI 生成剧本

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.2 |
| **用例名称** | AI 生成剧本 |
| **前置条件** | TC-5.1 通过 |
| **请求** | `POST /api/v1/projects/{project_id}/script/generate` |
| **请求体** | `{ "prompt": "一个赛博朋克蓝发少女从公寓醒来，穿过霓虹街道去买咖啡，遇到了一个神秘机器人。包含3个场景，每个场景2-3个镜头。", "mode": "generate" }` |
| **预期响应** | `code=0`，`data.content` 非空 |
| **验证点** | 1. `data.content` 为非空字符串<br>2. 内容包含场景/镜头相关的结构化文本<br>3. 文本长度 > 100 字符 |
| **超时** | 最长等待 60 秒 |
| **后续操作** | 保存剧本（`PUT /api/v1/projects/{project_id}/script`，body: `{ "content": <生成的内容> }`） |

### TC-5.3 AI 拆解镜头

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.3 |
| **用例名称** | AI 拆解剧本为场景和镜头 |
| **前置条件** | TC-5.2 通过，剧本已保存 |
| **请求** | `POST /api/v1/projects/{project_id}/script/parse` |
| **预期响应** | `code=0`，返回 `ParseResult` |
| **验证点** | 1. `data.scenes` 数组长度 >= 1<br>2. 每个 scene 包含 `title` 和 `shots` 数组<br>3. 每个 shot 包含 `title`、`narration`、`action_description`<br>4. `data.warnings` 为数组（可能包含未匹配的角色/场景警告）<br>5. 获取项目详情 `GET /api/v1/projects/{project_id}`，验证 `scenes` 已填充 |
| **后续操作** | 保存所有 `scene_id` 和 `shot_id` |

### TC-5.4 镜头关联角色并引用角色视图图片

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.4 |
| **用例名称** | 给镜头关联角色并引用具体角色视图图片 |
| **前置条件** | TC-5.3 通过（有 `shot_id`），TC-2.1 + TC-2.2 通过（有 `character_id` 和角色视图 `view_id`） |
| **请求** | `PUT /api/v1/projects/{project_id}/shots/{shot_id}` |
| **请求体** | `{ "character_ids": [<character_id>], "ref_character_view_id": <view_id> }` |
| **预期响应** | `code=0`，返回 `ShotOut` |
| **验证点** | 1. `data.characters` 数组长度 >= 1<br>2. `data.characters[0].id == <character_id>`<br>3. `data.characters[0].name == "测试角色"`<br>4. `data.ref_character_view_id == <view_id>`（引用了具体的角色视图图片） |

### TC-5.5 场景关联场景环境并引用场景图片

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.5 |
| **用例名称** | 给场景关联场景环境，给镜头引用具体场景图片 |
| **前置条件** | TC-5.3 通过（有 `scene_id`、`shot_id`），TC-3.1 + TC-3.2 通过（有 `environment_id` 和 `environment_image_id`） |
| **步骤一** | `PUT /api/v1/projects/{project_id}/scenes/{scene_id}`，请求体：`{ "environment_id": <environment_id> }` |
| **步骤一验证** | 1. `data.environment` 不为 `null`<br>2. `data.environment.id == <environment_id>`<br>3. `data.environment.name == "赛博朋克街道"` |
| **步骤二** | `PUT /api/v1/projects/{project_id}/shots/{shot_id}`，请求体：`{ "ref_environment_image_id": <environment_image_id> }` |
| **步骤二验证** | 1. `data.ref_environment_image_id == <environment_image_id>`（引用了具体的场景图片） |

### TC-5.6 生成镜头图片

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.6 |
| **用例名称** | 参考角色视图和场景图片生成镜头图片 |
| **前置条件** | TC-5.4 + TC-5.5 通过，镜头已关联角色（含角色视图引用）和场景（含场景图片引用） |
| **请求** | `POST /api/v1/projects/{project_id}/shots/{shot_id}/generate` |
| **请求体** | `{ "task_type": "image" }` |
| **预期响应** | `code=0`，返回 `GenerateResponse` |
| **验证点** | 1. `data.task_id` 为正整数<br>2. `data.task_type == "image"`<br>3. `data.status == "pending"`<br>4. `data.credits_cost == 2`<br>5. 轮询 `GET /api/v1/tasks/{task_id}`，直到 `status == "completed"`<br>6. 获取项目详情，目标 shot 的 `images` 数组长度 >= 1<br>7. 每张图片的 `image_url` 可访问<br>8. 确认 shot 的 `ref_environment_image_id` 和 `ref_character_view_id` 均有值（生图时参考了具体的场景图片和角色视图） |
| **超时** | 最长等待 180 秒 |
| **后续操作** | 保存 `image_id`（用于后续锁定） |

### TC-5.7 锁定镜头图片

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-5.7 |
| **用例名称** | 锁定选中的镜头图片 |

### TC-5.8 基于镜头图片生成视频

| **验证点** | 1. 所有分镜视频文件存在且可播放<br>2. 拼接后的视频文件大小 > 0<br>3. 拼接后的视频可正常播放<br>4. 视频顺序与剧本场景/镜头顺序一致 |

## 七、测试执行流程总览

```
TC-1.1 登录
  │
  ├─→ TC-2.1 创建角色
  │     ├─→ TC-2.2 生成角色视图
  │     ├─→ TC-2.3 新增角色视图
  │     └─→ TC-2.4 删除角色视图
  │
  ├─→ TC-3.1 创建场景环境
  │     ├─→ TC-3.2 生成场景图片
  │     ├─→ TC-3.3 新增场景图片
  │     └─→ TC-3.4 删除场景图片
  │
  ├─→ TC-4.1 创建风格
  │     └─→ TC-4.2 生成风格参考图
  │
  └─→ TC-5.1 创建项目（依赖 TC-4.1 的 style_id）
        ├─→ TC-5.2 AI 生成剧本
        ├─→ TC-5.3 AI 拆解镜头
        ├─→ TC-5.4 镜头关联角色并引用角色视图（依赖 TC-2.1 + TC-2.2）
        ├─→ TC-5.5 场景关联场景环境并引用场景图片（依赖 TC-3.1 + TC-3.2）
        ├─→ TC-5.6 生成镜头图片
        ├─→ TC-5.7 锁定镜头图片
        ├─→ TC-5.8 基于镜头图片生成视频
        └─→ TC-6.1 导出 JSON
             ├─→ TC-6.2 导出 ZIP 下载
             └─→ TC-6.3 视频拼接产出
```

### 可并行执行的用例组

以下用例组之间**无依赖关系**，可并行执行以加速测试：

| 并行组 | 包含用例 | 说明 |
|--------|---------|------|
| **Group A** | TC-2.1 → TC-2.2 → TC-2.3 → TC-2.4 | 角色管理全流程 |
| **Group B** | TC-3.1 → TC-3.2 → TC-3.3 → TC-3.4 | 场景环境全流程 |
| **Group C** | TC-4.1 → TC-4.2 | 风格管理全流程 |
