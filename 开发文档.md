# CI.AI 详细开发文档

**基于产品文档版本：** 2026-04-27
**技术栈：** React 18 + TypeScript / FastAPI + Celery + PostgreSQL + Redis + MinIO

---

## 一、 数据模型设计

### 1.1 User（用户）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 用户 ID |
| username | VARCHAR(64) | UNIQUE, NOT NULL | 用户名 |
| password_hash | VARCHAR(256) | NOT NULL | 密码哈希（bcrypt） |
| role | VARCHAR(16) | NOT NULL, DEFAULT 'user' | 角色：admin / user |
| credits | INTEGER | NOT NULL, DEFAULT 1000 | 积分余额 |
| avatar_url | VARCHAR(512) | NULLABLE | 头像地址 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 更新时间 |

**索引：** `idx_user_username` ON (username)

---

### 1.2 Project（项目）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 项目 ID |
| name | VARCHAR(128) | NOT NULL | 项目名称 |
| description | TEXT | NULLABLE | 项目描述 |
| creator_id | BIGINT | FK → User.id, NOT NULL | 创建者 |
| style_id | BIGINT | FK → Style.id, NULLABLE | 引用的风格预设（项目级） |
| shots_per_image | INTEGER | NOT NULL, DEFAULT 2 | 每个分镜生成的候选图片数量 |
| status | VARCHAR(16) | NOT NULL, DEFAULT 'draft' | 项目状态：draft / in_progress / completed |
| locked_by | BIGINT | FK → User.id, NULLABLE | 当前持有编辑锁的用户 ID，NULL 表示无锁 |
| lock_heartbeat | TIMESTAMP | NULLABLE | 锁的最后心跳时间 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 更新时间 |

**索引：**
- `idx_project_creator` ON (creator_id)
- `idx_project_style` ON (style_id)

### 1.14 ER 关系总览

```
User ──1:N──> Project
User ──1:N──> Character
User ──1:N──> Environment
User ──1:N──> Style

Project ──1:1──> Script
Project ──N:1──> Style          (项目级风格引用)
Project ──1:N──> Scene

Scene ──N:1──> Environment      (场景引用全局场景资产)
Scene ──1:N──> Shot

Environment ──1:N──> EnvironmentImage  (场景资产包含多张图片)

Shot ──N:N──> Character         (通过 ShotCharacter 关联表)
Shot ──N:1──> EnvironmentImage  (ref_environment_image_id 引用具体场景图片)
Shot ──N:1──> CharacterView     (ref_character_view_id 引用具体角色视图)
Shot ──1:N──> ShotImage
Shot ──1:N──> GenerationTask
Shot ──1:1──> ShotImage         (locked_image_id 锁定引用)
```

## 二、 后端 API 接口文档

**基础路径：** `/api/v1`
**认证方式：** JWT Bearer Token（登录后获取，附带在 `Authorization: Bearer <token>` 请求头中）
**通用响应格式：**

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**通用错误码：**

| code | 说明 |
|------|------|
| 0 | 成功 |
| 401 | 未认证 |
| 403 | 无权限 / 编辑锁被占用 |
| 404 | 资源不存在 |
| 409 | 冲突（如锁竞争） |
| 422 | 参数校验失败 |
| 429 | 积分不足 |
| 500 | 服务端错误 |

### 2.1 认证模块 (Auth)

#### POST /auth/register
注册新用户。

**请求体：**
```json
{
  "username": "string (3-64字符)",
  "password": "string (6-128字符)"
}
```

**响应 data：**
```json
{
  "id": 1,
  "username": "alice",
  "role": "user",
  "credits": 1000
}
```

#### POST /auth/login
用户登录，返回 JWT Token。

**请求体：**
```json
{
  "username": "string",
  "password": "string"
}
```

**响应 data：**
```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "alice",
    "role": "user",
    "credits": 1000
  }
}
```

#### GET /auth/me
获取当前登录用户信息。

**响应 data：** 同 register 返回的 user 对象。

### 2.3 项目编辑锁 (Project Lock)

#### POST /projects/{project_id}/lock
尝试获取项目编辑锁。

**响应 data（成功）：**
```json
{
  "locked": true,
  "locked_by": { "id": 1, "username": "alice" },
  "lock_heartbeat": "2026-04-27T10:30:00Z"
}
```

**响应（锁被占用，code=409）：**
```json
{
  "code": 409,
  "message": "编辑锁已被占用",
  "data": {
    "locked_by": { "id": 2, "username": "bob" },
    "lock_heartbeat": "2026-04-27T10:28:00Z"
  }
}
```

#### POST /projects/{project_id}/lock/heartbeat
续期编辑锁心跳。前端每 30 秒调用一次。

#### DELETE /projects/{project_id}/lock
释放编辑锁（自己释放或管理员强制释放）。

### 3.2 核心模块实现细节

#### 3.2.1 认证与权限 (`services/auth_service.py`)

- **密码存储：** 使用 `bcrypt` 对密码进行哈希。
- **JWT 令牌：** 使用 `python-jose` 签发 JWT，默认有效期 24 小时。Token 中包含 `user_id` 和 `role`。
- **依赖注入：** 在 `dependencies.py` 中定义 `get_current_user` 依赖，解析请求头中的 JWT 并返回用户对象。`require_admin` 依赖用于管理员权限校验。

#### 3.2.2 编辑锁 (`services/lock_service.py`)

- **锁存储：** 利用 Project 表的 `locked_by` 和 `lock_heartbeat` 字段，配合 Redis 分布式锁保证并发安全。
- **获取锁：** Redis `SET NX` 原子操作 + 数据库更新。如果 `lock_heartbeat` 超过 60 秒未续期，视为锁过期可被抢占。
- **心跳续期：** 前端每 30 秒调用一次 heartbeat 接口，更新 `lock_heartbeat` 时间戳和 Redis TTL。
- **释放锁：** 清除 `locked_by`，删除 Redis key。管理员可跳过持有者校验直接释放。
- **编辑锁校验依赖：** `require_edit_lock` 依赖函数，在写操作路由中使用，校验当前用户是否持有目标项目的编辑锁。

#### 3.2.3 剧本拆解 (`services/script_service.py`)

- **AI 拆解流程：**
  1. 从 `SystemSettings` 读取文本 API 配置。
  2. 构造拆解 Prompt：将剧本文本 + 预设指令发送给文本生成 API，要求返回结构化 JSON。
  3. 解析返回的 JSON，创建 Scene 和 Shot 记录。
  4. **资产匹配：** 遍历拆解出的角色名，在 Character 表中进行模糊匹配（`ILIKE`）。对场景同理在 Environment 表中匹配。
  5. 匹配成功则自动填入关联 ID，匹配失败则记录 warning 返回前端提示。
  6. 将拆解状态 `script.parsed` 标记为 `True`。

#### 3.2.4 Prompt 拼装器 (`ai/prompt_builder.py`)

- **拼装规则：** 按固定顺序拼接各维度的 Prompt 片段：
  1. **风格：** 从 Project 引用的 Style 中取 `prompt`。
  2. **场景：** 从 Shot 所在 Scene 引用的 Environment 中取 `prompt` + `description`。
  3. **角色：** 遍历 Shot 关联的 Character，取 `visual_prompt`，多个角色用逗号分隔。
  4. **动作：** 取 Shot 的 `action_description`。
  5. **镜头语言：** 将 `camera_angle` 映射为英文描述（如 `closeup` → `extreme close-up shot`）。
- **输出：** 返回拼装后的完整 Prompt 字符串和各组成部分的分解。

#### 3.2.5 生成任务调度 (`services/generation_service.py` + `tasks/`)

- **积分校验：** 发起任务前检查用户积分余额，不足则返回 429。扣费在任务创建时即刻执行（预扣），任务失败后不退还（防刷）。
- **任务入队：** 创建 `GenerationTask` 记录后，调用 `celery_task.delay()` 将任务推送到对应队列。
- **并发控制：** 为每类 API 创建独立的 Celery 队列（`queue_image`, `queue_video`, `queue_audio`），并通过 Celery worker 的 `--concurrency` 参数控制并发数。并发数从 `SystemSettings` 动态读取。
- **重试机制：** Celery task 装饰器设置 `max_retries=2, default_retry_delay=10`。每次重试前更新 `GenerationTask.retry_count`。
- **结果回写：** 任务成功后更新 `GenerationTask.status` 和 `result_url`，通过 WebSocket 推送 `task_completed` 消息。图片生成任务会同时创建 `ShotImage` 记录。

#### 3.2.6 AI API 适配层 (`ai/`)

- **抽象基类 `BaseAdapter`：**
  ```
  class BaseAdapter:
      endpoint: str
      api_key: str
      timeout: int
      async def generate(self, prompt, **kwargs) -> dict
      async def check_status(self, task_id) -> dict  # 针对异步 API
  ```
- 每种适配器（TextAdapter / ImageAdapter / VideoAdapter / AudioAdapter）继承基类，实现具体的 HTTP 请求逻辑。
- 适配器在初始化时从 `SystemSettings` 读取配置，支持运行时动态刷新。

#### 3.2.7 文件存储 (`services/storage_service.py`)

- 使用 `boto3` 与 MinIO 交互（兼容 S3 API）。
- **Bucket 规划：**
  - `ci-ai-uploads`：用户上传的原始文件
  - `ci-ai-generated`：AI 生成的图片/视频/音频
  - `ci-ai-exports`：导出的 ZIP 包
- 上传后返回文件的公共访问 URL（MinIO 配置 public-read 策略或使用预签名 URL）。

#### 3.2.8 导出打包 (`services/export_service.py`)

- 遍历项目下所有 Shot，收集 `video_url`、`audio_url`、`locked_image` URL。
- 从 MinIO 下载所有文件到临时目录，按 `场景名/镜头名/` 组织目录结构。
- 生成元数据 JSON 文件（包含每个镜头的时间戳、台词、字幕等信息）。
- 使用 `zipfile` 打包为 ZIP，上传至 `ci-ai-exports` bucket，返回下载 URL（有效期 24 小时）。

#### 3.2.9 WebSocket 管理 (`websocket/manager.py`)

- 维护一个 `dict[project_id, list[WebSocket]]` 的连接池。
- 用户连接时验证 JWT Token（通过 query parameter 传递）。
- 支持按 project_id 广播消息。
- Celery 任务通过 Redis Pub/Sub 将事件发布到 WebSocket 管理器，再由管理器分发到对应项目的所有连接。

### 4.3 核心页面与组件实现

#### 4.3.1 项目列表页 (`ProjectList`)

- **布局：** 卡片网格布局，每个项目显示名称、风格、状态、创建者、创建时间。
- **操作：** 新建项目（弹窗表单：名称、描述、选择风格、配置分镜图片数量）、点击进入项目详情、删除项目。
- **数据：** 调用 `GET /projects` 接口，使用 `usePagination` Hook 管理分页。

#### 4.3.2 项目详情页 (`ProjectDetail`) — 核心页面

**整体布局：三栏式**

```
┌──────────────────────────────────────────────────────┐
│  LockBar (编辑锁状态 + 积分余额)                       │
├──────────┬──────────────────────────────┬─────────────┤
│          │                              │             │
│  Script  │       Canvas                 │   Preview   │
│  Panel   │   (场景容器 + 分镜卡片)        │   Player    │
│          │                              │             │
│  (可折叠) │   上下滚动                     │  (可折叠)    │
│          │                              │             │
├──────────┴──────────────────────────────┴─────────────┤
│  底部工具栏：批量生成按钮 | 导出按钮 | 生成进度总览       │
└──────────────────────────────────────────────────────┘
```

- **进入页面时：**
  1. 调用 `GET /projects/{id}` 获取完整项目数据，写入 `projectStore`。
  2. 建立 WebSocket 连接 (`useWebSocket`)，监听生成进度和锁状态变更。
  3. 如果需要编辑，点击"进入编辑"按钮调用 `POST /lock` 获取锁，成功后激活编辑能力。
  4. `useLock` Hook 启动 30 秒定时器自动续期心跳。

- **ScriptPanel（剧本面板）：**
  - 显示剧本文本的编辑器（编辑模式下可编辑，只读模式下不可编辑）。
  - "AI 生成"按钮：弹窗输入创作提示/关键词，调用 `POST /script/generate`。
  - "智能拆解"按钮：调用 `POST /script/parse`，拆解完成后画布区域自动渲染出场景和镜头卡片。拆解结果中的 `warnings`（未匹配的角色/场景）以 Alert 形式展示，引导用户跳转到资产中心构建。
  - 面板可折叠收起，给画布更多空间。

- **Canvas（画布区域）：**
  - 纵向滚动布局，不支持缩放。
  - 使用 `dnd-kit` 的 `DndContext` + `SortableContext` 实现分镜卡片的拖拽排序。
  - 拖拽排序后自动调用 `PUT /shots/reorder` 同步到后端。

- **SceneGroup（场景容器组件）：**
  - 每个场景渲染为一个可折叠的卡片分组，标题显示场景名称和引用的场景资产。
  - 场景未关联资产时显示警告标识，点击可跳转到资产中心。
  - 支持"新增镜头"、"删除场景"操作按钮（仅编辑模式可见）。

- **ShotCard（分镜卡片组件）：**
  - 紧凑卡片布局，显示：镜头标题、缩略图（候选图片/锁定图片）、状态徽标、关联角色头像列表。
  - 点击展开 `ShotEditor` 弹窗进行详细编辑。
  - 状态徽标颜色映射：pending=灰色, images_generated=蓝色, image_locked=绿色, video_generated=紫色, completed=金色。
  - 每张卡片右侧显示快捷操作：生成图片、生成视频、删除。

- **ShotEditor（分镜编辑器弹窗）：**
  - 弹窗/抽屉式编辑面板，包含：
    - **文本区域：** 旁白、台词、字幕的多行输入框。
    - **动作描述：** 单行/多行输入框。
    - **镜头语言：** 下拉选择（特写 / 中景 / 远景 / 俯拍 / 摇移 / 跟拍等）。
    - **角色关联：** 多选下拉，从全局角色资产中选择。未构建的角色显示提示。
    - **Prompt 预览：** 实时调用 `GET /shots/{id}/prompt` 展示拼装后的完整 Prompt。
  - 编辑内容变更后调用 `PUT /shots/{id}` 保存。

- **ImageSelector（候选图片选择器）：**
  - 在 ShotCard 展开后显示，网格展示该分镜的所有候选图片。
  - 每张图片可放大预览。
  - 点击"锁定"按钮调用 `POST /shots/{id}/lock-image`，锁定后图片边框高亮，其他图片置灰。
  - 已锁定状态下可切换选中其他图片重新锁定。

- **PreviewPlayer（预览播放器）：**
  - 右侧面板，可折叠。
  - 支持按顺序连续播放所有已生成视频的分镜。
  - 播放列表按场景 > 镜头排序，显示镜头标题和时长。
  - 播放到某个镜头时，画布中对应的 ShotCard 高亮联动。

- **LockBar（编辑锁状态栏）：**
  - 顶部固定栏，显示当前项目编辑锁状态。
  - 无锁时显示"进入编辑"按钮。
  - 自己持有锁时显示"编辑中"+ 绿色指示灯 +"退出编辑"按钮。
  - 他人持有锁时显示"xxx 正在编辑"+ 锁定图标。管理员额外显示"强制接管"按钮。
  - 右侧显示当前用户积分余额 (`CreditsDisplay`)。

#### 4.3.3 资产库页面 (`AssetLibrary`)

- **布局：** 顶部 Tab 切换三个子页面：角色 / 场景 / 风格。
- **列表页：** 卡片网格布局，每个资产显示名称、缩略图、创建时间。支持关键词搜索和分页。
- **新建操作：** 弹窗表单 + 文件上传组件。

- **CharacterDetail（角色详情页）：**
  - 左侧：角色基本信息编辑（名称、描述、视觉 Prompt、音色配置）。
  - 右侧：角色视图图片网格（最多 20 张）。
  - "上传种子图"按钮：上传参考图片。
  - "生成视图"按钮：弹窗选择生成数量和视角类型，调用 `POST /characters/{id}/generate-views`。
  - 每张视图图片支持删除。视图数量达到 20 张时"生成视图"按钮置灰。

- **EnvironmentDetail（场景资产详情页）：**
  - 编辑场景名称、描述、提示词。
  - 上传或 AI 生成场景底图。

- **StyleDetail（风格详情页）：**
  - 编辑风格名称、提示词。
  - 上传风格参考图片。

#### 4.3.4 系统设置页 (`Settings`)

- **布局：** 表单页面，按 API 类型分为四组（文本 / 图片 / 视频 / 音频）。
- **每组配置项：** API 端点、API 密钥（密码输入框）、并发数（数字输入框，最小 1）、超时时间（秒）。
- 保存按钮调用 `PUT /settings`。
- 仅管理员可编辑 API 配置，普通用户可查看并发配额。

### 4.5 WebSocket 集成 (`hooks/useWebSocket.ts`)

```typescript
function useWebSocket(projectId: number) {
  // 1. 建立连接：ws://host/ws/projects/{projectId}?token=xxx
  // 2. 监听消息类型，分发到对应 Store：
  //    - task_progress  → generationStore.onTaskProgress
  //    - task_completed → generationStore.onTaskCompleted + projectStore.onTaskCompleted
  //    - task_failed    → generationStore.onTaskFailed (显示错误通知)
  //    - lock_changed   → projectStore.onLockChanged
  //    - content_changed → projectStore.fetchProject (重新拉取最新数据)
  // 3. 断线自动重连（指数退避，最大间隔 30 秒）
  // 4. 组件卸载时关闭连接
}
```

```typescript
function useLock(projectId: number) {
  // 1. acquireLock(): 调用 POST /projects/{id}/lock
  //    - 成功 → projectStore.isEditing = true

  // 2. startHeartbeat(): setInterval 每 30 秒调用 POST /lock/heartbeat

  // 5. 组件卸载时自动释放锁
```

### 4.7 关键交互流程

#### 流程一：剧本拆解

```
用户输入/粘贴剧本文本
  → 点击"保存剧本" → PUT /script
  → 点击"智能拆解" → POST /script/parse
  → 前端接收拆解结果
    → 画布区域渲染 SceneGroup + ShotCard
    → 如有未匹配的角色/场景，弹出 Alert 提示
    → 用户可手动新增/删除场景和镜头
```

#### 流程二：分镜图片生成与锁定

```
用户点击 ShotCard 上的"生成图片"
  → 前端检查积分余额 ≥ 2 × shots_per_image
  → POST /shots/{id}/generate { task_type: "image" }
  → ShotCard 状态变为"生成中"（loading 动画）
  → WebSocket 推送 task_completed
    → ShotCard 显示候选图片缩略图
  → 用户点击某张图片 → "锁定"
    → POST /shots/{id}/lock-image { image_id: xxx }
    → ShotCard 状态变为 image_locked，显示锁定图标
```

#### 流程三：批量视频生成

```
用户点击底部工具栏"批量生成视频"
  → 前端校验：所有分镜是否都已锁定图片
    → 未锁定的镜头标红提示
  → POST /projects/{id}/generate-all { task_type: "video" }
  → 后端按队列顺序逐个处理
  → WebSocket 实时推送每个 Shot 的进度
  → 画布中 ShotCard 状态实时更新
```

#### 流程四：导出

```
用户点击"导出压缩包"
  → POST /projects/{id}/export
  → 后端异步打包（可通过 WebSocket 推送进度）
  → 返回 download_url
  → 前端自动触发浏览器下载
```
