# 产品需求文档 (PRD)：AI 视频全流程自动化创作平台

**产品名称：** CI.AI
**产品定位：** 一站式、资产驱动型 AI 视频生产工作台。通过将角色、场景、风格资产化，实现从剧本到成品视频的工业化流水线作业，旨在将 AI 视频制作效率提升 10 倍以上。

---

## 一、 核心功能模块划分

### 0. 项目管理中心 (Project Hub)
系统的顶层组织单元，所有创作活动都在"项目"内进行。
*   **项目 (Project)：** 一个完整的视频创作任务即为一个项目。每个项目拥有独立的剧本、场景列表和分镜序列。
*   **项目与资产的关系：** 项目通过引用的方式使用全局资产（角色、场景、风格），同一资产可被多个项目复用。
*   **项目与剧本的关系：** 一个项目对应一个剧本，一对一绑定。
*   **项目内部层级：** 项目 > 场景 (Scene) > 分镜 (Shot)，一个项目包含多个场景，一个场景包含多个分镜。

### 1. 全局资产中心 (Global Asset Library)
这是系统的"大脑"，负责存储和管理所有可复用的创作要素，确保视频的一致性。资产是全局的、跨项目共享的。
*   **角色资产模块：**
    *   **数字档案：** 存储角色名称、性格描述、视觉特征 Prompt。
    *   **多维视图：** 用户可上传种子图作为参考，基于图片生成 API 生成角色的多视角参考图（正/侧/背、不同表情、不同动作等）。每个角色最多支持 **20 张**视图图片。生成不满意可重新生成。
    *   **音色克隆/绑定：** 关联角色的 TTS (Text-to-Speech) 音色或上传录音素材。
*   **场景资产模块：**
    *   **环境设定：** 存储场景的基础描述词（如：光影、天气、建筑风格）。
    *   **场景底图：** 存储该场景空间结构图，用于维持不同分镜间的空间一致性。
*   **风格资产模块：**
    *   **风格资产库：** 全局管理视觉风格定义。每个风格由**风格提示词**和**参考图片**组成。
    *   **项目级引用：** 风格作为**项目级的全局属性**，在创建/编辑项目时选定，不在单个镜头级别设置。项目内所有分镜共享同一风格。

### 2. 智能剧本与结构化引擎 (Script & Parser)
将线性文字转化为生产指令。剧本一旦确认后不支持版本回退。
*   **AI 剧本助手：** 支持通过关键词扩写剧本、修改对白、调整节奏。
*   **自动拆解（Parsing）：** AI 自动将剧本识别为"场景 > 镜头"的树状结构：
    *   **拆解产出：** 多个场景，每个场景包含多个镜头。
    *   **场景级：** 每个场景自动生成场景描述提示词，并尝试匹配全局资产库中已有的场景资产。若资产库中无匹配的场景资产，则仅保留提示词，**提示用户前往资产中心手动构建场景**后再引用。
    *   **镜头级：** 每个镜头自动识别出现的角色，并尝试匹配已有的角色资产。若角色未提前构建，则**提示用户前往资产中心构建角色**。每个镜头可选择镜头语言（特写、俯冲、摇移等），系统自动拼装最终生图/生视频的提示词。
*   **用户修正：** 拆解结果支持人工调整——可新增或删除场景，也可在场景内新增或删除单个镜头。

### 3. 生产工作流画布 (Production Canvas)
视频构建的主战场，采用"场景包裹分镜"的嵌套布局。画布支持**上下滑动**浏览，**不支持缩放**。分镜之间的转场效果暂不考虑。
*   **场景容器（Scene Group）：** 批量管理属于同一场景的所有分镜。
*   **分镜编辑器（Shot Editor）：**
    *   **内容区：** 编辑旁白、台词、字幕。
    *   **资产槽位（Asset Slots）：** 快速勾选/拖拽关联角色和场景。
    *   **智能 Prompt 拼装器：** 自动融合"风格+场景+角色+视角+动作"生成最终绘图指令。
    *   **拖拽排序：** 分镜卡片支持拖拽调整顺序。
*   **图片选定与锁定：**
    *   每个分镜生成多张候选图片（数量在新建项目时配置，默认 **2 张**）。
    *   用户从候选图片中选择满意的一张并**锁定**。锁定后的图片用于后续图转视频。
    *   **未锁定图片的分镜无法启动图转视频生成。**
*   **生成控制器：** 单个分镜的音频、图片、视频生成按钮及进度显示。

### 4. 并行渲染与队列管理器 (Render Manager)
*   **批量处理引擎：** 一键启动"全剧本生成"，后台自动按队列顺序处理音频同步、图像渲染和视频扩帧。
*   **并发配额管理：** 在系统 Settings 页面中展示并允许用户修改各类 API 的并发数，默认均为 **1 并发**。
*   **失败重试机制：** 生成任务失败后自动重试 **2 次**。自动重试耗尽后仍失败，用户可**手动触发重试**。
*   **状态监控：** 实时显示每个分镜的生成状态（等待中、生成中、已完成、失败）。页面刷新后分镜状态持久保留。
*   **积分体系：**
    *   每个用户初始积分为 **1000**。
    *   一次图片生成消耗 **2 积分**，一次视频生成消耗 **10 积分**。
    *   积分不足时无法发起生成任务，需管理员充值或系统扩展付费机制。

### 5. 导出与资产打包 (Export & Packaging)
*   **一键打包：** 自动将所有分镜视频、对应音频、字幕文件分类存放，打包为 ZIP 供下载。
*   **工程导出：** 支持导出带有时间戳的元数据文件，方便二次剪辑。

## 二、 依托于本系统的 AI 视频生产流程

使用本系统后，你的视频创作将从“碎片化搜索”变为“线性工业生产”：

### 第一步：资产预制 (Asset Pre-production)
1.  **定义风格：** 选择或创建一个视觉风格（如：3D 渲染风格）。
2.  **创建角色：** 输入描述生成角色主图，利用系统工具一键生成角色的 8 个视角参考图，并选定配音。
3.  **创建场景：** 输入场景关键词（如：赛博朋克街道），生成并锁定场景基础图。

### 第二步：剧本导入与拆解 (Scripting & Parsing)
1.  **输入剧本：** 粘贴你的故事文本或者基于AI生成一个故事文本。
2.  **AI 拆单：** 点击“智能拆解”，系统自动在画布上生成：
    *   场景 1：街道。包含分镜 1.1、1.2、1.3。
    *   场景 2：室内。包含分镜 2.1、2.2。
3.  **自动匹配：** 系统根据剧本文字，自动把“角色A”和“场景1”填入对应的分镜槽位。

### 第三步：精调与指令合成 (Prompt Integration)
1.  **调整分镜：** 在分镜卡片中，简单修改动作描述（如：从“走路”改为“奔跑”）。
2.  **视角选定：** 为每个分镜快速点击预设视角（如：特写、俯冲、摇移）。
3.  **预览 Prompt：** 系统自动合成最终的复杂 Prompt，无需手动输入。

### 第四步：生产 (Mass Production)
1.  **后台挂机：** 
    *   首先进行分镜图片的生成，每个分镜生成至少2张图片，供人工挑选
    *   随后根据分镜图片生成AI视频，视频自带音频，如果不带音频的话，也可以单独生成音频或者上传音频

### 第五步：收尾与导出 (Final Export)
1.  **快速预览：** 在右侧预览窗连续播放生成的片段，不满意的地方点击“重新生成”。
2.  **一键下载：** 点击“导出压缩包”，获取所有制作素材，进入后期剪辑软件合成。

## 三、 系统的核心价值（提速点）

1.  **减少重复输入：** 角色和场景只需设定一次，全局资产跨项目复用。
2.  **消除提示词焦虑：** 通过“资产槽位”自动组装 Prompt，用户无需学习复杂的提示词工程。
3.  **并行取代串行：** 在后台生成分镜 1 的视频时，用户可以同时处理分镜 10 的脚本，极大地压缩了等待时间。
4.  **视觉一致性保证：** 通过“资产库”强制约束，彻底解决 AI 视频中常见的“分镜换脸”、“场景变样”的问题。

## 四、 技术架构与选型

### 1. 整体架构
采用前后端分离架构，前端为 SPA 单页应用，后端提供 RESTful API + WebSocket 实时通信。

### 2. 前端技术栈
*   **框架：** React 18 + TypeScript
*   **状态管理：** Zustand（轻量、简洁，适合中型项目）
*   **UI 组件库：** Ant Design 5.x（丰富的企业级组件，加速开发）
*   **画布/拖拽：** 基于 dnd-kit 实现分镜卡片的拖拽排序
*   **实时通信：** WebSocket（用于协作状态同步、生成进度推送）
*   **构建工具：** Vite

### 3. 后端技术栈
*   **框架：** Python FastAPI（异步高性能，天然适合 AI 任务调度与长时间运行的生成任务）
*   **任务队列：** Celery + Redis（管理异步生成任务的调度与并发控制）
*   **数据库：** PostgreSQL（关系型数据，存储项目、资产、分镜等结构化数据）
*   **对象存储：** MinIO 或兼容 S3 协议的存储服务（存储图片、音频、视频等大文件）
*   **缓存/消息：** Redis（任务队列 broker、锁机制、会话缓存）

### 4. AI 模型对接
系统采用**可配置的 API 适配层**设计，不绑定特定模型供应商。用户可在系统后台自行配置以下四类 AI 能力的 API 端点与密钥：
*   **文本生成 API：** 用于剧本扩写、智能拆解、Prompt 合成。
*   **音频生成 API：** 用于 TTS 语音合成、音色克隆。
*   **图片生成 API：** 用于角色参考图、场景底图、分镜画面生成。
*   **视频生成 API：** 用于图生视频（Image-to-Video）扩帧渲染。

每类 API 支持配置并发限制、超时时间、重试策略，由后台队列管理器统一调度。

### 5. 部署方案
*   **容器化部署：** Docker + Docker Compose，支持一键本地部署。
*   **可选云部署：** 后续可扩展至 Kubernetes 集群部署。

## 五、 数据模型与实体关系

### 1. 核心实体

| 实体 | 说明 | 归属关系 |
|------|------|----------|
| **User（用户）** | 系统使用者，拥有账号、权限和积分余额（初始 1000） | — |
| **Project（项目）** | 一个完整的视频创作任务，包含项目配置（如每个分镜生成图片数量，默认 2） | 由 User 创建 |
| **Script（剧本）** | 项目的文字脚本，不支持版本回退 | 与 Project 一对一 |
| **Scene（场景组）** | 项目内的一个场景段落，引用全局场景资产 | 属于 Project，一对多 |
| **Shot（分镜）** | 场景内的一个镜头单元，包含候选图片和锁定状态 | 属于 Scene，一对多 |
| **ShotImage（分镜候选图片）** | 分镜生成的候选图片，支持锁定选中 | 属于 Shot，一对多 |
| **Character（角色资产）** | 全局角色定义，最多 20 张视图图片 | 全局，可被多个 Project 引用 |
| **Environment（场景资产）** | 全局场景/环境定义 | 全局，可被多个 Project 引用 |
| **Style（风格预设）** | 全局视觉风格定义，由风格提示词 + 参考图片组成 | 全局，可被多个 Project 引用 |
| **GenerationTask（生成任务）** | 记录每次图片/视频/音频的生成请求及状态 | 属于 Shot |
| **SystemSettings（系统设置）** | 各类 API 的并发配额、端点配置等 | 全局 |

### 2. 关键关系

```
User ──1:N──> Project
Project ──1:1──> Script
Project ──1:N──> Scene ──1:N──> Shot
Shot ──1:N──> ShotImage    （一个分镜生成多张候选图片，其中一张可被锁定）
Shot ──1:N──> GenerationTask（一个分镜可发起多次生成任务）

Shot ──N:N──> Character    （一个分镜可包含多个角色，一个角色可出现在多个分镜）
Scene ──N:1──> Environment （一个场景引用一个场景资产，一个场景资产可用于多个场景）
Project ──N:1──> Style     （一个项目选用一个风格预设（项目级），一个风格可被多个项目使用）
```

### 3. 资产的全局性
*   角色（Character）、场景（Environment）、风格（Style）存储在**全局资产库**中，不隶属于任何特定项目。
*   项目通过**引用关系**使用资产，修改资产会影响所有引用该资产的项目（需要在 UI 上做变更提示）。
*   **风格的特殊性：** 风格在项目级别绑定（而非镜头级别），项目内所有分镜共享同一风格。风格由提示词和参考图片组成。

## 六、 用户体系与协作机制

### 1. 用户账户
*   **登录认证：** 支持用户名/密码登录（后续可扩展 OAuth 第三方登录）。
*   **用户角色：** 初期仅区分管理员和普通用户两种角色。

### 2. 协作模型
系统面向**小团队协作**场景设计（个位数用户），采用**乐观读、悲观写**策略：
*   **查看（只读）：** 无锁。任何有权限的用户可随时打开项目查看内容和生成进度，不会阻塞其他人。
*   **编辑（写入）：** 有锁。同一时刻只有一个用户可获得项目的编辑锁，获取编辑锁后方可修改剧本、分镜、资产关联等。
*   **锁机制细节：**
    *   用户点击"进入编辑"时尝试获取锁，若锁已被占用则提示当前编辑者信息。
    *   锁具有**心跳续期**机制，用户离开页面或长时间无操作后自动释放锁。
    *   管理员可**强制释放**他人持有的锁（用于异常情况处理）。
*   **实时状态同步：** 通过 WebSocket 向所有查看者推送编辑变更和生成进度更新。
